# 재생

> **참조 소스**: `src/playback/PlaybackApp.cc`, `src/playback/Player.cc`, `lib/Streamer.cc`, `lib/StreamExtractor.cc`, `lib/SearchDb.cc`

---

## 1. 재생 엔진 개요

```
검색 요청 (시간/이벤트)
    │
    ▼
SearchDb → IMF 파일 위치 확인
    │
    ▼
Streamer: IMF 파일에서 스트림 추출
    │
    ▼
StreamExtractor: 채널별 비디오/오디오 분리
    │
    ▼
[HAL] Video Decoder: H.264/H.265 디코딩
    │
    ▼
[HAL] Display: 디코딩 프레임 출력
    │
    ▼
AudioPlayer: 오디오 재생 (동기)
```

---

## 2. 재생 모드

| 모드 | 설명 |
|------|------|
| **시간 검색 재생** | 특정 시각의 녹화 영상 재생 |
| **이벤트 재생** | 이벤트 발생 시점으로 이동 후 재생 |

---

## 3. 재생 제어

### 3.1 IPC 메시지

| 메시지 | 설명 |
|--------|------|
| **IM_PLAYBACK_CMD** | 재생 명령 (PLAY, PAUSE, STOP, SEEK 등) |
| **IM_PLAYBACK_VIEW** | 재생 뷰 전환 |
| **IM_SET_PLAYBACK_PARAM** | 재생 파라미터 설정 |
| **IM_TIMESEARCH_PLAYBACK** | 시간 기반 재생 시작 |
| **IM_REQ_PLAYBACK** | 재생 요청 |

### 3.2 재생 명령

| 명령 | 설명 |
|------|------|
| PLAY | 재생 시작/재개 |
| PAUSE | 일시 정지 |
| STOP | 재생 중지 |
| SEEK | 특정 시점으로 이동 |
| FAST_PLAY | 빠른 재생 (2x, 4x, 8x) |
| SLOW_PLAY | 느린 재생 (0.5x, 0.25x) |
| STEP | 프레임 단위 이동 |

### 3.3 뷰 모드

| 모드 | 설명 |
|------|------|
| SINGLE | 단일 채널 전체 화면 |
| PIP | Picture-in-Picture |
| SPLIT | 다중 채널 분할 |

채널 선택: 비트마스크 기반

## 4. 검색 기능

### 4.1 시간 차트

월/일/시/분 단위의 녹화 존재 여부 차트:
- TimeSearchChart: 시간 범위별 데이터 존재 표시
- 이벤트 타입별 색상 구분

### 4.2 이벤트 검색

EventDb 기반:
- 시간 범위 필터
- 이벤트 타입 필터 (알람, 모션, AI, 시스템)
- 채널 필터
- 운행/주차 모드 필터

### 4.3 검색 결과

```
검색 조건 → SearchDb 쿼리
    │
    ▼
IMF 파일 목록 + 오프셋
    │
    ▼
시간 차트 데이터 (UI 표시용)
    │
    ▼
이벤트 목록 (시각, 타입, 채널)
```

## 5. [HAL] 비디오 디코딩

### 5.1 디코딩 흐름

```
IMF 파일 → Streamer
    │
    ▼
스트림 데이터 추출 (채널별)
    │
    ▼
암호화된 경우 → 복호화 (AES-128)
    │
    ▼
[HAL] 디코더 입력 (H.264/H.265 패킷)
    │
    ▼
디코딩 프레임 (NV12)
    │
    ▼
[HAL] Display 레이어에 출력
```

### 5.2 디코딩 관리

- DecodeManager: 채널별 디코더 인스턴스 관리
- 동시 다채널 디코딩 지원
- 스냅샷용 디코딩: 재생 중 특정 프레임 JPEG 캡처

---

## 6. 재생 동기화

### 6.1 AV 동기화

- 비디오 프레임 타임스탬프 기반
- 오디오는 비디오에 동기
- 빠른 재생 시 오디오 무음

### 6.2 멀티채널 동기화

- 동일 IMF 파일 내 채널 간 타이밍 정보 참조
- 세그먼트 인덱스 기반 프레임 위치 계산

---

## 7. 원격 재생

HTTP 서비스를 통한 원격 재생:

```
HTTP 클라이언트 → /playback?time=...&ch=...
    │
    ▼
Network 프로세스: Streamer 생성
    │
    ▼
IMF → 스트림 추출 → HTTP 응답 스트리밍
```
